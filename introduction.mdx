# Welcome to Soraka

Soraka is a full-stack web application designed for managing hospital appointments and medical consultations. The platform enables patients to browse available doctors, view open appointment slots, and book consultations, while providing doctors and administrators with powerful tools to manage the entire appointment lifecycle.

<Note>
This project is currently under active development. Some features are still being built and refined.
</Note>

## What is Soraka?

Soraka provides a complete appointment management solution for healthcare facilities. Patients can discover available doctors by specialty, view their schedules, and reserve appointments. Medical professionals can create and manage appointment slots, while administrators maintain full control over users, doctors, specialties, and system configuration.

## Role-Based Access System

Soraka implements a hierarchical role-based access control (RBAC) system with three distinct user roles:

```text
ADMIN > MEDICO > PACIENTE
```

This hierarchy means that an **ADMIN** inherits all permissions from MEDICO and PACIENTE roles, while a **MEDICO** inherits permissions from the PACIENTE role.

| Role | Permissions |
|---|---|
| **ADMIN** | Complete system control: manage users, doctors, specialties, and all appointments |
| **MEDICO** | Create, modify, delete, and view appointments. Register new users |
| **PACIENTE** | View available appointments, book consultations, and cancel own appointments |

## Technology Stack

Soraka is built with modern, production-ready technologies:

### Backend

- **Java 21** with **Spring Boot 3.5.9**
- **Spring Security + JWT** for stateless authentication
- **Spring Data JPA / Hibernate** for database access
- **MySQL 8.0** as the relational database
- **Spring Mail** for email verification

### Frontend

- **React 18.2** with **Vite** build system
- **React Router 7.13** for navigation
- **Tailwind CSS** with **Material Tailwind** components
- **Heroicons** and **Lucide React** for icons
- **jwt-decode** for token management

### Infrastructure

- **Docker** and **Docker Compose** for containerization
- **phpMyAdmin** for database administration
- Timezone configured for **Europe/Madrid**

## Security & Authentication

Soraka implements a robust JWT-based authentication flow:

1. Administrators register users whose accounts start in an **inactive** state
2. A **confirmation email** with a unique token is sent to the user
3. Users activate their accounts by clicking the email link
4. Upon activation, users can log in and receive a **JWT token valid for 1 hour**
5. All protected endpoints require the token in the `Authorization: Bearer <token>` header

The JWT token includes user name, email, and role as additional claims for authorization decisions.

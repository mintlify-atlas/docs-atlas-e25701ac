## Overview

Soraka requires several environment variables to configure database connections, JWT authentication, and email services. These variables should be defined in a `.env` file in the project root.

## Required Variables

### Database Configuration

<ParamField path="MYSQL_ROOT_PASSWORD" type="string" required>
  Root password for the MySQL database. Used by both the MySQL container and Spring Boot application.
  
  **Example**: `mysecurepassword123`
  
  <Warning>
    Use a strong password for production deployments. Avoid simple passwords like "root" or "password".
  </Warning>
</ParamField>

<ParamField path="SPRING_DATASOURCE_USERNAME" type="string" required>
  Username for Spring Boot to connect to MySQL. Typically set to `root` for development.
  
  **Default**: `root`
  
  **Example**: `root`
</ParamField>

<ParamField path="SPRING_DATASOURCE_PASSWORD" type="string" required>
  Password for Spring Boot database connection. Should match `MYSQL_ROOT_PASSWORD` when using root user.
  
  **Example**: `mysecurepassword123`
</ParamField>

### JWT Authentication

<ParamField path="SECRET_KEY" type="string" required>
  Secret key used to sign JWT tokens. Must be kept confidential and should be a long, random string.
  
  **Used in**: `application.properties` as `jwt.secret=${SECRET_KEY}`
  
  **Example**: `myverysecretjwtkey2024!@#$%^&*()`
  
  <Info>
    Generate a secure random key using: `openssl rand -base64 32`
  </Info>
</ParamField>

### Email Configuration

<ParamField path="MAIL_USERNAME" type="string" required>
  Gmail account username for sending email notifications (account activation, password reset, etc.).
  
  **Used in**: `application.properties` as `spring.mail.username=${MAIL_USERNAME}`
  
  **Example**: `noreply@yourhospital.com`
</ParamField>

<ParamField path="MAIL_PASSWORD" type="string" required>
  Gmail app password for email authentication.
  
  **Used in**: `application.properties` as `spring.mail.password=${MAIL_PASSWORD}`
  
  <Warning>
    Use a Gmail App Password, not your regular Gmail password. Generate one at: https://myaccount.google.com/apppasswords
  </Warning>
  
  **Example**: `abcd efgh ijkl mnop` (16-character app password)
</ParamField>

## Environment File Template

Create a `.env` file in the project root with the following content:

```env .env
# Database Configuration
MYSQL_ROOT_PASSWORD=your_secure_password_here
SPRING_DATASOURCE_USERNAME=root
SPRING_DATASOURCE_PASSWORD=your_secure_password_here

# JWT Configuration
SECRET_KEY=your_jwt_secret_key_here

# Email Configuration (Gmail)
MAIL_USERNAME=your_email@gmail.com
MAIL_PASSWORD=your_gmail_app_password_here
```

## Configuration Flow

Environment variables are used in multiple layers of the application:

### 1. Docker Compose Layer

The `docker-compose.yml` file references variables from `.env`:

```yaml docker-compose.yml
services:
  db:
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
  
  backend:
    env_file:
      - .env
    environment:
      SPRING_DATASOURCE_USERNAME: ${SPRING_DATASOURCE_USERNAME}
      SPRING_DATASOURCE_PASSWORD: ${SPRING_DATASOURCE_PASSWORD}
```

### 2. Spring Boot Layer

The `application.properties` file uses environment variables:

```properties application.properties
# Database
spring.datasource.username=root
spring.datasource.password=root

# JWT
jwt.secret=${SECRET_KEY}

# Email
spring.mail.username=${MAIL_USERNAME}
spring.mail.password=${MAIL_PASSWORD}
```

<Info>
  Docker Compose overrides the hardcoded values in `application.properties` with environment variables from the `.env` file.
</Info>

## Variable Reference Table

| Variable | Type | Required | Default | Used By | Purpose |
|----------|------|----------|---------|---------|----------|
| `MYSQL_ROOT_PASSWORD` | string | Yes | - | MySQL, Spring Boot | Database root password |
| `SPRING_DATASOURCE_USERNAME` | string | Yes | `root` | Spring Boot | Database connection username |
| `SPRING_DATASOURCE_PASSWORD` | string | Yes | - | Spring Boot | Database connection password |
| `SECRET_KEY` | string | Yes | - | Spring Boot | JWT token signing key |
| `MAIL_USERNAME` | string | Yes | - | Spring Boot | SMTP username for emails |
| `MAIL_PASSWORD` | string | Yes | - | Spring Boot | SMTP password for emails |

## Security Best Practices

<Warning>
  **Never commit the `.env` file to version control.** Add it to `.gitignore`:
  
  ```bash .gitignore
  .env
  .env.local
  .env.*.local
  ```
</Warning>

### Password Requirements

- **Database passwords**: Minimum 12 characters, mix of letters, numbers, and symbols
- **JWT secret**: Minimum 32 characters, highly random
- **Gmail app password**: 16 characters (generated by Google)

### Generating Secure Values

```bash
# Generate a random JWT secret
openssl rand -base64 32

# Generate a random password
openssl rand -base64 24
```

## Validation

Verify your environment variables are loaded correctly:

```bash
# Check Docker Compose configuration (with variable substitution)
docker-compose config

# Check if backend can read variables
docker-compose logs backend | grep -i "error"
```

## Gmail Configuration

To enable email functionality:

<Steps>
  <Step title="Enable 2-Factor Authentication">
    Go to your Google Account settings and enable 2FA:
    https://myaccount.google.com/security
  </Step>

  <Step title="Generate App Password">
    Navigate to App Passwords:
    https://myaccount.google.com/apppasswords
    
    - Select "Mail" as the app
    - Select "Other" as the device
    - Name it "Soraka Hospital System"
    - Copy the 16-character password
  </Step>

  <Step title="Update .env file">
    Add the credentials to your `.env` file:
    
    ```env
    MAIL_USERNAME=your_email@gmail.com
    MAIL_PASSWORD=abcd efgh ijkl mnop
    ```
  </Step>

  <Step title="Restart the backend">
    ```bash
    docker-compose restart backend
    ```
  </Step>
</Steps>

## Email Server Configuration

The application uses Gmail's SMTP server with the following settings (configured in `application.properties`):

- **Host**: `smtp.gmail.com`
- **Port**: `587`
- **Authentication**: Enabled
- **STARTTLS**: Enabled

<Info>
  To use a different email provider, modify the `spring.mail.*` properties in `backend/src/main/resources/application.properties` and rebuild the container.
</Info>

## Troubleshooting

### "Authentication failed" error

**Cause**: Invalid Gmail credentials or app password.

**Solution**: 
1. Verify 2FA is enabled on your Google account
2. Generate a new app password
3. Ensure no spaces in the password in `.env` file

### "Access denied for user" database error

**Cause**: Mismatch between `MYSQL_ROOT_PASSWORD` and `SPRING_DATASOURCE_PASSWORD`.

**Solution**: Ensure both variables have the same value when using the root user.

### JWT token validation fails

**Cause**: `SECRET_KEY` is missing or changed after tokens were issued.

**Solution**: 
1. Verify `SECRET_KEY` is set in `.env`
2. Restart the backend container
3. Users may need to log in again if the key changed

## Production Deployment

For production environments:

<Warning>
  - Use a secrets management service (AWS Secrets Manager, HashiCorp Vault, etc.)
  - Rotate credentials regularly
  - Use different credentials for each environment (dev, staging, prod)
  - Enable SSL/TLS for database connections
  - Consider using a dedicated SMTP service (SendGrid, AWS SES, etc.)
</Warning>

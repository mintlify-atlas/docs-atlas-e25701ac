---
title: "Delete Appointment"
api: "DELETE /api/citas/{id}"
description: "Permanently delete an appointment from the system"
---

## Authorization

This endpoint requires one of the following authorities:
- `MEDICO`
- `ADMIN`

<ParamField path="id" type="integer" required>
  The unique identifier of the appointment to delete
</ParamField>

<RequestExample>

```bash cURL
curl -X DELETE https://api.soraka.hospital/api/citas/42 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const appointmentId = 42;
const response = await fetch(`https://api.soraka.hospital/api/citas/${appointmentId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});

if (response.ok) {
  console.log('Appointment deleted successfully');
}
```

```java Java
Long appointmentId = 42L;
HttpRequest request = HttpRequest.newBuilder()
  .uri(URI.create("https://api.soraka.hospital/api/citas/" + appointmentId))
  .header("Authorization", "Bearer YOUR_TOKEN")
  .DELETE()
  .build();
HttpResponse<Void> response = client.send(request, HttpResponse.BodyHandlers.discarding());
```

</RequestExample>

## Notes

- Only doctors and administrators can delete appointments
- This operation is permanent and cannot be undone
- Deleting an appointment removes it completely from the system
- Consider using the [cancel endpoint](/api/appointments/cancel) instead if you want to keep the appointment record but mark it as cancelled
- If you need to free up a time slot, cancelling returns the appointment to `DISPONIBLE` status

<Warning>
  Deleting an appointment is a permanent action. If you want to preserve the appointment history, use the cancel endpoint instead.
</Warning>

---
title: "List Available Appointments"
api: "GET /api/citas/disponibles"
description: "Retrieve all available appointment slots with future dates"
---

## Authorization

This endpoint requires authentication with any of the following authorities:
- `PACIENTE`
- `MEDICO`
- `ADMIN`

<RequestExample>

```bash cURL
curl -X GET https://api.soraka.hospital/api/citas/disponibles \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.soraka.hospital/api/citas/disponibles', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const availableSlots = await response.json();
```

```java Java
HttpRequest request = HttpRequest.newBuilder()
  .uri(URI.create("https://api.soraka.hospital/api/citas/disponibles"))
  .header("Authorization", "Bearer YOUR_TOKEN")
  .GET()
  .build();
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
```

</RequestExample>

## Response Fields

<ResponseField name="id" type="integer" required>
  Unique identifier for the appointment slot
</ResponseField>

<ResponseField name="pacienteId" type="integer">
  Always `null` for available appointments
</ResponseField>

<ResponseField name="pacienteNombre" type="string">
  Always `null` for available appointments
</ResponseField>

<ResponseField name="medicoId" type="integer" required>
  Doctor's ID
</ResponseField>

<ResponseField name="medicoNombre" type="string" required>
  Doctor's full name
</ResponseField>

<ResponseField name="medicoEspecialidad" type="string" required>
  Doctor's medical specialty
</ResponseField>

<ResponseField name="fechaHora" type="string" required>
  Appointment date and time in ISO-8601 format (always in the future)
</ResponseField>

<ResponseField name="estado" type="enum" required>
  Always `DISPONIBLE` for this endpoint
</ResponseField>

<ResponseField name="motivo" type="string">
  Always `null` for available appointments
</ResponseField>

## Notes

- Only returns appointments with status `DISPONIBLE`
- Only returns appointments with future dates
- Patients can use this endpoint to find slots they can book
- Doctors and admins can use this to view available capacity

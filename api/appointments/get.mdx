---
title: "Get Appointment"
api: "GET /api/citas/{id}"
description: "Retrieve a specific appointment by ID"
---

## Authorization

This endpoint requires one of the following authorities:
- `MEDICO`
- `ADMIN`

<ParamField path="id" type="integer" required>
  The unique identifier of the appointment
</ParamField>

<RequestExample>

```bash cURL
curl -X GET https://api.soraka.hospital/api/citas/42 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const appointmentId = 42;
const response = await fetch(`https://api.soraka.hospital/api/citas/${appointmentId}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const appointment = await response.json();
```

```java Java
Long appointmentId = 42L;
HttpRequest request = HttpRequest.newBuilder()
  .uri(URI.create("https://api.soraka.hospital/api/citas/" + appointmentId))
  .header("Authorization", "Bearer YOUR_TOKEN")
  .GET()
  .build();
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
```

</RequestExample>

## Response Fields

<ResponseField name="id" type="integer" required>
  Unique identifier for the appointment
</ResponseField>

<ResponseField name="pacienteId" type="integer">
  Patient's user ID (null if appointment is available)
</ResponseField>

<ResponseField name="pacienteNombre" type="string">
  Patient's full name (null if appointment is available)
</ResponseField>

<ResponseField name="medicoId" type="integer" required>
  Doctor's ID
</ResponseField>

<ResponseField name="medicoNombre" type="string" required>
  Doctor's full name
</ResponseField>

<ResponseField name="medicoEspecialidad" type="string" required>
  Doctor's medical specialty
</ResponseField>

<ResponseField name="fechaHora" type="string" required>
  Appointment date and time in ISO-8601 format
</ResponseField>

<ResponseField name="estado" type="enum" required>
  Appointment status: `DISPONIBLE`, `CONFIRMADA`, `REALIZADA`, or `CADUCADA`
</ResponseField>

<ResponseField name="motivo" type="string">
  Reason for the appointment (max 255 characters)
</ResponseField>

## Notes

- Only doctors and administrators can retrieve individual appointments
- Returns appointments in any status
- Use this endpoint to get detailed information about a specific appointment

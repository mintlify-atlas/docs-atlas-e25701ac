---
title: "Update Appointment"
api: "PATCH /api/citas/{id}"
description: "Partially update an existing appointment"
---

## Authorization

This endpoint requires one of the following authorities:
- `MEDICO`
- `ADMIN`

<ParamField path="id" type="integer" required>
  The unique identifier of the appointment to update
</ParamField>

<RequestExample>

```bash cURL
curl -X PATCH https://api.soraka.hospital/api/citas/42 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "fechaHora": "2026-03-20T11:00:00",
    "motivo": "Consulta de seguimiento actualizada"
  }'
```

```javascript JavaScript
const appointmentId = 42;
const response = await fetch(`https://api.soraka.hospital/api/citas/${appointmentId}`, {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    fechaHora: '2026-03-20T11:00:00',
    motivo: 'Consulta de seguimiento actualizada'
  })
});
const updatedAppointment = await response.json();
```

```java Java
Long appointmentId = 42L;
```

## Request Body

All fields are optional. Only provide the fields you want to update.

<ParamField body="fechaHora" type="string">
  New appointment date and time in ISO-8601 format (e.g., `2026-03-20T11:00:00`)
</ParamField>

<ParamField body="estado" type="enum">
  New appointment status: `DISPONIBLE`, `CONFIRMADA`, `REALIZADA`, or `CADUCADA`
</ParamField>

<ParamField body="motivo" type="string">
  Updated reason for the appointment (max 255 characters)
</ParamField>

<ParamField body="medicoId" type="integer">
  Change the assigned doctor
</ParamField>

## Response Fields

<ResponseField name="id" type="integer" required>

<ResponseField name="pacienteId" type="integer">

<ResponseField name="pacienteNombre" type="string">

<ResponseField name="medicoNombre" type="string" required>
  Doctor's full name
</ResponseField>

<ResponseField name="medicoEspecialidad" type="string" required>
  Doctor's medical specialty
</ResponseField>

<ResponseField name="fechaHora" type="string" required>
  Appointment date and time in ISO-8601 format
</ResponseField>

<ResponseField name="estado" type="enum" required>
  Updated appointment status
</ResponseField>

<ResponseField name="motivo" type="string">
  Updated reason for the appointment
</ResponseField>

## Notes

- Only doctors and administrators can update appointments
- This is a partial update (PATCH) - only send the fields you want to change
- Changing the doctor (`medicoId`) may be subject to availability constraints
- Business rules and validation are enforced by the service layer

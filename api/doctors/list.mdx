---
title: "List All Doctors"
api: "GET /api/medicos"
description: "Retrieve a complete list of all registered doctors with full details including usuario and especialidad relationships."
---

## Authentication

This endpoint requires authentication and ADMIN role authorization.

<ParamField header="Authorization" type="string" required>
  Bearer token with ADMIN privileges
</ParamField>

## Response

Returns an array of doctor objects with complete information.

<ResponseField name="id" type="long">
  Unique identifier for the doctor
</ResponseField>

<ResponseField name="usuarioId" type="long">
  ID of the associated usuario (user account)
</ResponseField>

<ResponseField name="nombreUsuario" type="string">
  Full name of the doctor from usuario entity
</ResponseField>

<ResponseField name="emailUsuario" type="string">
  Email address from usuario entity
</ResponseField>

<ResponseField name="especialidadId" type="long">
  ID of the doctor's medical specialty
</ResponseField>

<ResponseField name="nombreEspecialidad" type="string">
  Name of the medical specialty
</ResponseField>

<ResponseField name="urlFoto" type="string">
  URL to the doctor's profile photo
</ResponseField>

<RequestExample>
```bash cURL
curl -X GET "https://api.soraka.com/api/medicos" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```


## Entity Relationships

The response includes denormalized data from related entities:

- **Medico**: Core doctor entity with `id` and `urlFoto`
- **Usuario**: One-to-one relationship providing user account details (`usuarioId`, `nombreUsuario`, `emailUsuario`)
- **Especialidad**: Many-to-one relationship providing specialty information (`especialidadId`, `nombreEspecialidad`)

Each Medico must be associated with exactly one Usuario and one Especialidad.

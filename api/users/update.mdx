---
title: "Update User"
api: "PATCH /api/usuarios/{id}"
description: "Partially update an existing user's information"
---

<Warning>
  This endpoint requires **ADMIN** role. Only administrators can update user information.
</Warning>

## Authorization

This endpoint requires authentication with a valid JWT token and ADMIN role.

```bash
Authorization: Bearer <token>
```

## Path Parameters

<ParamField path="id" type="long" required>
  The unique identifier of the user to update
</ParamField>

## Request Body

All fields are optional. Only include fields you want to update.

<ParamField body="nombre" type="string" optional>
  User's full name (2-50 characters)
</ParamField>

<ParamField body="email" type="string" optional>
  User's email address (must be valid email format and unique)
</ParamField>

<ParamField body="password" type="string" optional>
  New password for the user (minimum 6 characters)
</ParamField>

<ParamField body="rol" type="enum" optional>
  User's role. Possible values:
  - `PACIENTE` - Patient
  - `MEDICO` - Doctor
  - `ADMIN` - Administrator
</ParamField>

<ParamField body="isActivo" type="boolean" optional>
  Whether the user account is active. Set to `false` to deactivate a user
</ParamField>

## Response

Returns the updated user object with complete information.

<ResponseField name="id" type="long" required>
  Unique identifier for the user
</ResponseField>

<ResponseField name="nombre" type="string" required>
  User's full name
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address
</ResponseField>

<ResponseField name="rol" type="enum" required>
  User's role in the system
</ResponseField>

<ResponseField name="isActivo" type="boolean" required>
  Whether the user account is active
</ResponseField>

<ResponseField name="fechaRegistro" type="datetime" required>
  Registration timestamp (ISO 8601 format)
</ResponseField>

<RequestExample>
```bash cURL
curl --request PATCH \
  --url https://api.soraka.com/api/usuarios/1 \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "nombre": "Juan Carlos Pérez",
    "isActivo": true
  }'
```

```javascript JavaScript
const userId = 1;
const response = await fetch(`https://api.soraka.com/api/usuarios/${userId}`, {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer <token>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    nombre: 'Juan Carlos Pérez',
    isActivo: true
  })
});

const updatedUser = await response.json();
```

```python Python
import requests
import json

user_id = 1
headers = {
    'Authorization': 'Bearer <token>',
    'Content-Type': 'application/json'
}

data = {
    'nombre': 'Juan Carlos Pérez',
    'isActivo': True
}

response = requests.patch(
    f'https://api.soraka.com/api/usuarios/{user_id}',
    headers=headers,
    data=json.dumps(data)
)
updated_user = response.json()
```

## Notes

- This is a partial update endpoint (PATCH). You only need to send the fields you want to update.
- Email addresses must be unique across the system.
- Password updates will be automatically hashed before storage.
- Business validation rules are applied according to the service layer logic.
